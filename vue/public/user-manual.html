基于大语言模型的函数级代码测试用例生成系统用户手册

本手册用于指导系统用户完成账号登录、模型配置、函数信息维护、需求规格说明书录入、测试用例生成、自动评估、再次生成、历史记录导出等操作。 手册内容按照系统实际使用流程编写，适合作为日常使用说明、课程验收材料和系统演示辅助文档。

一、手册使用说明

本系统主要面向教师、学生、管理员以及测试相关使用者。系统围绕“函数级代码测试用例自动生成”这一核心场景设计，支持根据函数源码或需求规格说明书自动构造测试用例，并对生成结果进行质量评估。

推荐使用顺序：登录系统 → 配置模型 → 维护函数或需求信息 → 进入用例生成 → 查看评估结果 → 必要时再次生成 → 在生成记录中导出结果。

二、系统概述

系统以大语言模型为基础，针对单个函数或明确的需求规格内容进行测试分析与用例生成。生成后，系统自动调用评估模型，从正常路径覆盖、边界条件覆盖、异常分支覆盖和语法规范性等维度对测试用例进行分析，并返回综合评分和改进建议。

系统名称 基于大语言模型的函数级代码测试用例生成系统
核心目标 提高函数级测试用例编写效率，辅助用户快速获得结构化、可读性较强的测试用例结果。
主要输入 函数名称、所属类名、函数代码、输入说明、输出说明、备注信息，或需求规格说明书相关内容。
主要输出 结构化测试用例、自动评估结果、综合评分、改进建议、历史生成记录、Excel 导出文件。
支持语言 测试用例支持中文或英文输出，评估语言随当前输出语言同步变化。

三、角色与权限说明

超级管理员 拥有最高权限，可查看并维护管理员信息、用户信息、模型配置、函数信息、需求规格说明书以及生成记录。
管理员 可维护自己及所属用户可见的数据，通常负责模型配置、函数管理、需求规格说明书管理和用户管理。
普通用户 可使用用例生成、查看生成记录、维护本人可见的函数和需求信息，使用已配置的模型进行生成与评估。
不同角色在页面菜单和数据可见范围上存在差异。若发现某些数据不可见，优先检查当前登录账号角色及数据归属关系。

四、系统模块说明

系统首页展示系统概况、统计信息及图表内容，用于快速了解当前系统数据情况。
用例生成选择模型、生成策略和输出语言，输入或带入函数/需求信息，完成测试用例生成、自动评估和再次生成。
生成记录分页查看历史生成内容、评估分数与时间信息，并支持单条导出为 Excel 文件。
函数管理对函数名称、所属类名、源码、输入输出说明和备注进行新增、编辑、删除与跳转生成。
需求规格说明书管理维护需求名称、模块、需求描述、输入输出说明等内容，支持从需求出发生成测试用例。
模型配置新增和维护模型名称、接口地址、密钥、温度系数、最大 Token 及启用状态，并选择默认评估模型。
账号管理管理员信息和用户信息维护，便于区分不同角色与数据权限。

五、详细操作流程

1. 登录系统

  1. 打开系统登录页面，输入账号与密码。
  2. 登录成功后进入系统首页。
  3. 根据当前角色显示对应菜单项。

2. 配置生成模型与默认评估模型

  1. 进入 模型配置 页面。
  2. 点击 新增模型，填写模型名称、API 地址、API Key、温度系数、最大 Token、状态等信息。
  3. 保存后，将需要使用的模型状态设置为 启用。
  4. 在页面中选择一个已启用模型作为 默认评估模型。
生成模型和评估模型可以不是同一个。生成时由用户在用例生成页选择模型，评估时默认调用在模型配置页中设置的评估模型。

3. 维护函数信息

  1. 进入 函数管理 页面。
  2. 点击 新增函数。
  3. 依次填写函数名称、所属类名、语言、输入说明、输出说明、备注、函数代码等信息。
  4. 保存后可在列表中查看、修改、删除，或直接点击 生成测试用例 跳转到生成页。

4. 维护需求规格说明书

  1. 进入 需求规格说明书管理 页面。
  2. 新增需求并录入需求名称、模块、需求说明、输入与输出相关描述。
  3. 保存后可在列表中直接进入用例生成页面。

5. 生成测试用例

  1. 进入 用例生成 页面。
  2. 选择 模型、生成策略 和 输出语言。
  3. 确认当前页面中的函数或需求内容完整无误。
  4. 点击 生成测试用例。
  5. 右侧先显示生成结果，随后展示自动评估结果。

6. 再次生成

  1. 当右侧评估结果展示完成后，页面会显示 再次生成 按钮。
  2. 点击该按钮后，系统会结合首轮生成结果与评估建议重新生成测试用例。
  3. 再次生成结束后会重新触发自动评估。

六、生成与评估说明

1. 生成策略说明

普通生成直接根据输入内容构造测试用例，生成速度较快，适合一般函数场景。
链式分析生成在生成前引导模型对控制流、边界值、异常分支进行分析，适合逻辑较复杂的函数场景。

2. 输出语言说明

若用户在页面中选择中文，则测试用例和评估结果均以中文返回；若选择英文，则测试用例和评估结果均以英文返回。评估语言与输出语言保持一致，便于整体阅读和导出使用。

3. 评估维度说明

综合评分对当前测试用例整体质量的总结性打分。
正常路径覆盖是否覆盖主要业务流程和常规输入场景。
边界条件覆盖是否覆盖边界值、阈值点和临界输入。
异常分支覆盖是否覆盖空值、非法值、异常抛出等非正常流程。
语法规范性是否符合系统约定的输出结构与表述规范。
改进建议对缺失场景、表达问题和补充方向的说明，可作为再次生成依据。

七、生成记录与导出说明

  1. 进入 生成记录 页面，可分页查看历史生成数据。
  2. 页面中展示编号、来源类型、名称、模型名称、生成策略、生成结果、大模型评估、测试人员评估分数、生成时间等内容。
  3. 点击单条记录右侧的 导出 按钮，可将该条记录中的测试用例导出为 Excel 文件。
  4. 导出内容优先按“用例编号、测试目的、输入数据、执行步骤、预期结果”结构解析。若解析失败，则按原始文本逐行导出，保证导出功能可用。
生成记录页当前采用分页展示，便于集中浏览与单条导出。若历史记录较多，可结合分页器逐页查看。

八、页面顶部功能说明

用户手册用于在线查看当前系统手册，也可由浏览器执行下载或另存为操作。
主题切换用于切换系统配色主题，满足不同展示场景和个人使用习惯。
当前用户信息显示当前登录账号身份，用于区分超级管理员、管理员或普通用户。

九、常见问题与处理建议

1. 页面没有可选模型

请检查模型配置页面中是否已经新增模型，并将模型状态设置为启用。

2. 点击生成后没有返回结果

请检查模型接口地址、API Key、网络环境以及后端服务是否正常运行。

3. 自动评估失败

请检查默认评估模型是否已正确选择，模型是否启用，接口是否返回正常内容。

4. 再次生成按钮未显示

请确认当前页面已经完成自动评估并成功展示评估结果。若仍未显示，建议刷新页面并重新执行一次生成流程。

5. 导出内容格式不完整

系统会优先解析结构化测试用例文本。若模型输出格式差异较大，导出会退回为逐行导出模式。

6. 某些数据页面中不可见

请检查当前登录账号角色以及数据所属关系，不同角色能看到的数据范围不同。

十、使用建议

  • 在录入函数或需求时，输入说明、输出说明和备注越完整，生成结果通常越稳定。
  • 对于存在多重折扣、复杂条件判断、异常抛出逻辑的函数，更适合使用链式分析生成。
  • 在自动评估完成后，可根据改进建议决定是否执行再次生成，减少重复性操作。
  • 建议定期在生成记录中导出关键结果，便于后续整理实验数据或撰写论文材料。

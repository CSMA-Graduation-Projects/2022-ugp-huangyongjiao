# 毕业设计整改任务清单

## 必须立即修改

### 1. 云端后端不可验证

- 问题：前端静态页面可以访问，但后端 `https://two026-huangyongjiao-testgen.onrender.com` 登录接口检查时无响应或 SSL EOF，无法确认 `root/123456` 能登录。
- 修改目标：保证答辩现场可以完整演示登录、生成、评估、历史记录和分析流程。
- 建议做法：检查 Render 后端运行状态、环境变量、数据库连接、跨域配置；用 `root/123456` 实测登录，保存成功截图；准备本地运行或录屏兜底。
- 影响位置：部署 / PPT / 论文。

### 2. 补齐完整数据库初始化脚本

- 问题：本地脚本主要覆盖 `function_info`、`generate_record`，但代码还依赖 `admin`、`user`、`model_config`、`requirement_spec`。
- 修改目标：检查老师或同学拿到项目后能够从空数据库启动系统。
- 建议做法：新增完整 `schema.sql` 和 `data.sql`，包含所有表结构、字段、索引、初始管理员账号 `root/123456`、至少一个可用模型配置示例说明。
- 影响位置：代码 / 部署 / 查非项。

### 3. 补 README 和部署说明

- 问题：根目录未发现 README、部署说明、数据库说明、测试说明。
- 修改目标：让项目材料满足归档和答辩前检查要求。
- 建议做法：README 至少写清项目简介、目录结构、环境版本、数据库初始化、后端启动、前端启动、云端地址、账号密码、模型配置、常见问题。
- 影响位置：代码 / 部署 / 论文。

### 4. 给论文第 5 章补真实测试证据

- 问题：功能测试表全部为“通过”，但缺输入、输出、日志、记录编号、覆盖率结果和失败样例。
- 修改目标：让“系统测试与功能验证”能经得起追问。
- 建议做法：补 1 个完整生成案例、1 个再次生成案例、1 个 coverage.py 成功案例、1 个 SKIPPED/FAILED/TEST_FAILED 案例；写明函数名、模型、策略、记录 ID、评估分、覆盖率状态、语句覆盖率、分支覆盖率。
- 影响位置：论文 / PPT / AI味。

### 5. 补实验原始数据附件

- 问题：论文和 PPT 有模型对比、Token、延时、平均评分数据，但本地未找到原始导出文件。
- 修改目标：证明表 5.3、表 5.4 和 PPT 第 11 页数据来源真实。
- 建议做法：从系统导出生成记录 Excel，保留批量生成页面截图、用例分析页面截图、统计口径说明；文件放入合适目录并在论文中引用。
- 影响位置：论文 / PPT / 查非项 / AI味。

### 6. 修正论文封面、承诺书、授权书空白

- 问题：封面、诚信承诺书、版权授权书存在题目、签名、日期、职称、答辩组信息等空白。
- 修改目标：满足终稿格式和学校模板要求。
- 建议做法：按学院模板补齐所有必填字段；终稿导出 PDF 前逐页检查。
- 影响位置：论文 / 查非项。

### 7. 更新 Word 域并修正引用显示

- 问题：抽取文本中可见 `TOC`、`HYPERLINK`、`PAGEREF`、`REF _Ref...` 等域代码痕迹。
- 修改目标：最终 Word/PDF 中目录、交叉引用、参考文献引用正常显示。
- 建议做法：Word 中全选更新域，关闭域代码显示，重新生成目录；导出 PDF 后搜索 `REF`、`PAGEREF`、`HYPERLINK`，确保不存在。
- 影响位置：论文 / 查非项。

### 8. 修正表号和明显文字错误

- 问题：论文文字提到“表5.5”，但未识别到表 5.5；另有“字段。。”等小错误。
- 修改目标：消除低级规范问题。
- 建议做法：全文检查图表编号、交叉引用、双标点、错别字；重点查“表5.5”“。。”“paper/this paper”。
- 影响位置：论文 / 查非项。

## 建议尽快修改

### 9. 降低摘要和绪论的空泛感

- 问题：摘要和绪论表达完整但偏长、偏概念化，系统证据不够突出。
- 修改目标：让论文更像学生本人围绕实际系统完成的工作。
- 建议做法：摘要拆成背景、目标、方法、实现、验证 5 个层次；绪论中减少泛泛研究意义，增加本系统为什么选择函数级、为什么先做 Python coverage.py、为什么设计再次生成。
- 影响位置：论文 / AI味。

### 10. 增加代码级实现说明

- 问题：第 4 章已有模块说明，但接口路径、关键方法、字段流转还不够具体。
- 修改目标：让实现章节能直接对应代码。
- 建议做法：补充 `/generate/testcase`、`/generate/evaluate`、`/generate/batchExperiment`、生成记录保存字段、coverage.py 临时执行流程；点名 `GenerateService`、`CoverageService`、`GenerateRecordMapper` 的职责。
- 影响位置：论文 / 代码 / AI味。

### 11. 在 PPT 中补演示流程页

- 问题：PPT 有架构和页面展示，但缺少答辩现场操作路径。
- 修改目标：让老师快速看到“做了什么、怎么做、效果如何”。
- 建议做法：增加一页流程：登录 -> 模型配置 -> 函数管理 -> 用例生成 -> 自动评估 -> 再次生成 -> 历史记录 -> 用例分析。
- 影响位置：PPT / 部署。

### 12. PPT 实验页补数据来源

- 问题：PPT 第 10、11 页有模型对比结论，但未说明统计口径。
- 修改目标：避免实验结论被认为是直接写出来的。
- 建议做法：标注数据来源为生成记录表或导出 Excel，写明记录数量、筛选条件、统计时间；保留原始截图作为备用。
- 影响位置：PPT / 论文 / AI味。

### 13. 补测试说明或接口测试材料

- 问题：项目无自动化测试文件，也无手工测试说明。
- 修改目标：补足工程完整性。
- 建议做法：新增 `测试说明.md` 或 Postman 集合，至少包含登录、函数新增、模型配置、生成、评估、记录查询、导出、coverage.py 分支。
- 影响位置：代码 / 文档 / 查非项。

### 14. 明确系统安全边界

- 问题：代码中密码为明文比对，权限主要依赖请求参数和数据归属过滤，不适合夸大安全性。
- 修改目标：论文表述与系统真实能力一致。
- 建议做法：论文中只写“角色菜单控制和数据可见范围控制”，不要写“完善安全体系”；答辩时说明这是原型系统，后续可加 Token、密码哈希和后端鉴权。
- 影响位置：代码 / 论文 / PPT。

### 15. 核查参考文献格式和有效性

- 问题：参考文献数量多，部分较新文献的年份、卷期、URL、访问日期需核对。
- 修改目标：降低参考文献格式和真实性风险。
- 建议做法：逐条核对作者、题名、文献类型、期刊/会议、卷期页码、DOI/URL、访问日期；统一英文作者缩写格式。
- 影响位置：论文 / 查非项。

- 问题：已有 `vue/public/user-manual.html`，但不是根目录显式文档。
- 修改目标：方便归档审查。
- 建议做法：在 README 中链接用户手册，或补一个简版 `用户手册.md`，说明主要操作流程和截图。

- 问题：“生成、评估、记录、分析”“提供数据基础”等表述重复较多。
- 修改目标：降低模板化观感。
- 建议做法：保留首次完整说明，后续用具体字段、接口、截图、表格替代重复概括。
- 影响位置：论文 / AI味。

- 建议做法：明确当前只验证 Python unittest coverage.py；Java/JUnit/JaCoCo、复杂依赖、项目级上下文、模型结果人工复核属于后续工作。
- 影响位置：论文 / PPT / AI味。
